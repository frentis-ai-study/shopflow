# ADR-0005: 멀티셀러 주문을 판매자별 하위주문으로 분리

- **상태**: 수용(Accepted)
- **날짜**: 2026-07-16
- **관련**: 헌장 원칙 VII, spec FR-021·FR-022

## 맥락 및 문제

구매자는 한 번의 결제로 여러 판매자의 상품을 함께 살 수 있다. 그러나 배송·상태 관리·향후
정산은 판매자 단위로 이뤄져야 한다. 주문을 단일 덩어리로만 다루면 판매자별 하위주문·정산을
분리 추적할 수 없다.

## 검토한 선택지

1. **단일 주문 단위로만 관리** — 여러 판매자가 한 상태를 공유. 이행/정산 분리 불가.
2. **주문(결제 단위) + 판매자별 하위주문으로 분리** — 결제는 하나, 이행·상태는
   판매자별로 독립.

## 결정

**한 번의 결제로 생성된 주문은 판매자별 하위주문으로 분리하여 각각의 상태를
독립적으로 추적한다.** 결제·총액은 주문 단위로, 배송 상태 전이·정산 기준 금액은 하위주문으로
관리한다.

## 근거

- 판매자별 배송 처리·상태 관리가 자연스럽게 분리된다(FR-021).
- 배송완료 시각·판매자별 금액을 하위주문에 기록해 두면, 후속 정산 기능이 이 구조 위에서
  바로 동작한다(원칙 VII).
- 구매자는 하나의 주문으로 보되, 상세에서 판매자별 이행 상태를 구분해 볼 수 있다.

## 결과

- **좋은 점**: 판매자별 하위주문·정산 확장에 유연, 상태 전이 책임이 명확.
- **나쁜 점/트레이드오프**: 주문-하위주문의 2계층 모델로 데이터·조회가 다소 복잡해진다.
  주문 전체 상태는 하위주문 상태들의 집계로 표현한다.
