# ADR-0006: 금액은 고정소수점(정수 최소단위)으로 처리

- **상태**: 수용(Accepted)
- **날짜**: 2026-07-16
- **관련**: 헌장 원칙 III(데이터 무결성)

## 맥락 및 문제

가격·주문 총액·판매자별 금액 등 돈을 다룬다. 이진 부동소수점(float/double)은 반올림 오차가
누적되어 합계 불일치·정산 오류를 일으킬 수 있다.

## 검토한 선택지

1. **이진 부동소수점(float/double)** — 오차 누적, 금전 계산에 부적합.
2. **고정소수점(십진 Decimal 또는 정수 최소통화단위)** — 정확한 십진 연산.

## 결정

**모든 금액은 고정소수점으로 표현한다.** 저장·계산은 십진 고정소수점 타입 또는 정수 최소
통화 단위(예: "원" 정수)를 사용하고, 이진 부동소수점을 금액에 사용하지 않는다.

## 근거

- 반올림 오차 없는 정확한 합계·정산을 보장한다(원칙 III).
- 단일 통화(원) 기준이라 정수 최소단위 표현이 단순하고 안전하다(spec 가정).

## 결과

- **좋은 점**: 금액 합계·검증·정산의 정확성 보장, 테스트 결정성.
- **나쁜 점/트레이드오프**: 통화 단위·표현(정수 원 vs Decimal)의 구체 타입은 구현
  (`/speckit-plan`)에서 확정한다. 다중 통화 도입 시 단위·환율 정책을 별도로 다뤄야 한다(현재
  범위 밖).
