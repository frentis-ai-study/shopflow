# ADR-0004: 결제 전 재고 선점 + TTL 자동 해제

- **상태**: 수용(Accepted)
- **날짜**: 2026-07-16
- **관련**: 헌장 원칙 III·VII, spec FR-017·FR-020, SC-003·SC-005, 명확화(2026-07-16)

## 맥락 및 문제

한정 재고 상품을 여러 구매자가 동시에 결제하면 초과 판매가 발생할 수 있다. 또한 결제를
시작만 하고 완료하지 않는 사용자가 재고를 무한정 묶어 두면 정상 판매를 방해한다.

## 검토한 선택지

1. **결제 시점에만 재고 차감** — 결제 확정 순간 차감. 결제 진행 중 경합에 취약, 실패 시
   롤백 처리 복잡.
2. **결제 전 재고 선점 + 만료(TTL)** — 결제 시작 시 수량을 선점하고, 미완료 시 TTL 후
   자동 해제, 성공 시 확정 차감.
3. 무제한 오버셀 허용 후 사후 취소 — 커머스 신뢰에 부적합.

## 결정

**결제 확정 전에 주문 수량만큼 재고를 선점하고, 결제가 완료되지 않으면 선점 후 10분(TTL)
경과 시 자동 해제한다. 결제 성공 시 선점을 확정 차감으로 전환한다.** 선점 실패(재고 부족)
시 결제를 거절한다.

## 근거

- 선점으로 결제 진행 중 재고를 보장해 초과 판매를 방지한다(SC-003: 0건).
- TTL 자동 해제로 미결제 이탈이 재고를 영구히 묶지 않게 한다(SC-005: 100% 복원).
- TTL 10분은 구매자의 결제 완료 여유와 재고 회전 사이의 균형점으로 선택했다(명확화 결정).

## 결과

- **좋은 점**: 초과 판매 차단, 재고 묶임 방지, 결제 실패/이탈 경로가 명확.
- **나쁜 점/트레이드오프**: 선점 상태·만료를 관리하는 메커니즘(예: 만료 스윕/조건부 갱신)이
  필요하다. **동시 선점의 원자성 보장 방식(낙관적 vs 비관적 락)은 미결정이며 `/speckit-plan`
  에서 확정하는 후속 ADR로 남긴다.**
