# ADR-0002: 결제는 외부 위임, MVP는 모의 게이트웨이

- **상태**: 수용(Accepted)
- **날짜**: 2026-07-16
- **관련**: 헌장 원칙 II(결제·보안 신뢰성), spec FR-024, 명확화(2026-07-16)

## 맥락 및 문제

결제를 어떻게 처리할지 정해야 한다. 카드 정보 취급은 PCI-DSS 적용 범위를 키우고 사고 시
치명적이다. 동시에 MVP 단계에서 실 결제망 연동은 키·웹훅·심사 등 비용이 크다.

## 검토한 선택지

1. **자체 결제 처리(카드 정보 직접 취급)** — PCI-DSS 범위 급증, 위험 과다.
2. **외부 PSP 실연동(토스·아임포트 등)** — 안전하나 MVP에 과한 연동 비용.
3. **결제 어댑터 추상화 + MVP는 모의(시뮬레이션) 게이트웨이** — 인터페이스 뒤에 모의 구현,
   추후 실 PSP 구현체 교체.

## 결정

**결제는 외부에 위임하고 카드 원본 정보를 저장하지 않는다. 결제 어댑터를 추상화하되, MVP
에서는 성공/실패/멱등을 시뮬레이션하는 모의 게이트웨이를 기본 구현으로 사용한다.** 실 PSP
연동은 후속 이슈로 분리한다.

## 근거

- 카드 미저장·외부 위임으로 PCI-DSS 범위를 최소화한다(원칙 II).
- 어댑터 경계를 두면 실 PSP 교체 시 도메인 로직 변경이 없다.
- 모의 게이트웨이로 핵심 로직(재고 선점·멱등·주문 생성)을 실 결제망 없이 완결·검증할 수 있다.

## 결과

- **좋은 점**: 보안 위험 최소화, MVP 속도 확보, 결제 실패/멱등 경로를 결정적으로 테스트 가능.
- **나쁜 점/트레이드오프**: 실 결제망 특유의 상태(부분승인·웹훅 지연·망취소)는 이번 범위에서
  검증되지 않는다 → 실 PSP 연동 ADR/이슈에서 다룬다.
- 어댑터 인터페이스는 멱등키 전달을 계약에 포함한다([ADR-0003](./0003-idempotent-payment-exactly-once.md)).
