# ADR-0003: 멱등키로 정확히 한 번 결제

- **상태**: 수용(Accepted)
- **날짜**: 2026-07-16
- **관련**: 헌장 원칙 III·VII, spec FR-018, SC-004

## 맥락 및 문제

네트워크 재시도, 사용자의 이중 클릭, 타임아웃 후 재요청 등으로 동일 결제 요청이 여러 번
도달할 수 있다. 이때 이중 결제·중복 주문이 발생하면 실제 금전 피해로 이어진다.

## 검토한 선택지

1. **멱등키(idempotency key) 기반 처리** — 요청에 고유 키를 부여하고, 이미 처리된 키는
   기존 결과를 반환.
2. **분산 락으로 중복 방지** — 락 획득 실패 시 거절. 락 인프라·경계 복잡.
3. **중복 허용 후 사후 정리(reconciliation)** — 이중 결제가 잠깐이라도 발생, 커머스에 부적합.

## 결정

**결제·주문 생성을 멱등키로 처리하여 재시도·중복 요청에도 효과가 정확히 한 번만 발생하도록
한다.** 동일 멱등키의 재요청에는 최초 처리 결과를 그대로 반환하고, 결제·주문을 다시 만들지
않는다.

## 근거

- 이중 결제·중복 주문을 구조적으로 차단한다(SC-004: 0건).
- 멱등키는 결제 어댑터 계약의 일부로, 모의/실 PSP 모두에 일관 적용된다([ADR-0002](./0002-payment-external-delegation-mock-gateway.md)).
- 분산 락 대비 인프라 요구가 낮고, 결과 재현이 결정적이라 테스트가 쉽다.

## 결과

- **좋은 점**: 재시도 안전성, 결정적 동작, 실 PSP로 확장해도 동일 원칙 유지.
- **나쁜 점/트레이드오프**: 멱등키 저장소와 만료 정책이 필요하다. 키 생성 책임(클라이언트
  발급 vs 서버 발급)과 보존 기간은 구현(`/speckit-plan`)에서 확정한다.
